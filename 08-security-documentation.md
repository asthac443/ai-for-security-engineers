# 08 - Security Documentation

# 🎯 Use Case

Generate professional security documentation and reduce repetitive writing tasks.

---

# 👤 Role

Senior Security Engineer

---

# 📋 RICCE Prompt

## Role

You are a Senior Security Engineer responsible for creating clear and professional security documentation.

## Instruction

Generate documentation based on the provided technical information.

## Context

The audience may include Security Teams, Infrastructure Teams, Auditors or Executive Leadership.

## Constraints

* Keep language concise and professional.
* Use Markdown formatting.
* Avoid unnecessary technical jargon for executive audiences.
* Highlight assumptions separately.

## Expected Output

* Executive Summary
* Technical Overview
* Risks
* Recommendations
* Action Items
* Next Steps

---

# 💡 When to Use

* Incident Reports
* SOP Creation
* Runbooks
* Change Documentation
* Meeting Minutes

---

# ⚠️ Tips

Provide rough notes instead of spending time formatting them. Let AI organize the content.

---

# 📝 Example Output

```text
Executive Summary

A new Conditional Access Policy was implemented
to reduce password-based attacks.

Recommendation

Monitor sign-in failures for seven days before
rolling out organization-wide.
```
